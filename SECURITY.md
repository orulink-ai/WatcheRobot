# Security Policy

## Reporting

Please do not open public issues for vulnerabilities or leaked credentials.

Report security concerns to the project maintainers through the private channel agreed by the OrUlink team. If no channel is available, contact the repository owner privately on GitHub and request a secure reporting path.

## Sensitive Data Rules

Never commit:

- Wi-Fi SSIDs or passwords
- API keys, tokens, signing keys, certificates, or private keys
- local serial-port maps
- private machine paths
- customer or tester logs containing personal data

Release artifacts should be checked before upload and accompanied by checksums.
