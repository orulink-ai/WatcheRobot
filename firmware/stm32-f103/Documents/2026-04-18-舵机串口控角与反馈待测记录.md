# 2026-04-18 舵机串口控角与反馈待测记录

> **日期**: 2026-04-18  
> **项目**: watcheRobot_STM32  
> **主题**: 当前串口控角验证结果与反馈测试边界留档

## 1. 本轮目标

本轮目标只覆盖以下内容：

- 通过 `USART1 CLI` 直接设置舵机角度
- 通过 `ST-Link` 完成当前固件烧录
- 通过本地串口口径确认当前 CLI 交互行为

本轮**不**覆盖以下内容：

- 舵机反馈 ADC 引脚接入
- 反馈电压采样
- 电压到角度的映射标定
- 串口持续打印反馈角度

## 2. 当前已完成内容

### 2.1 串口控角

当前 CLI 已支持：

- `servo <angle>`
- `servo <id> <angle>`

说明：

- `servo <angle>` 默认控制 1 号舵机
- `servo <id> <angle>` 当前 `id` 支持 `1/2`
- 角度范围约束为 `0~180`

### 2.2 上电行为

当前固件上电后的舵机行为为：

- 默认归中
- 默认静止
- 不再上电自动往复运动

### 2.3 启动提示

当前固件在 `App_Init()` 结束后会通过 `USART1` 打印：

```text
=== STM32 CLI Ready ===
USART1 @ 115200 8N1
Commands: help, servo 90, servo 1 45, servo 2 135
>
```

### 2.4 烧录记录

本轮已通过 `ST-Link` 完成烧录，使用目标文件：

- `build/Debug/watcheRobot_STM32.elf`

烧录结果：

- 目标芯片识别正常
- 写入完成
- 写后校验通过
- 已执行 `MCU Reset`

## 3. 当前未完成内容

### 3.1 舵机反馈尚未进入主线

当前仓库**还没有**完成舵机反馈相关接入：

- `CubeMX` 尚未配置反馈使用的 `ADC`
- 反馈输入引脚尚未固定到当前主线配置
- 反馈采样代码尚未加入工程
- 电压到角度的映射函数尚未建立

### 3.2 本轮未做反馈实测

本轮没有进行反馈测试，原因是：

- 反馈舵机需要等明天到货后再做实物验证

因此，本轮提交只能确认：

- 串口控角链路可继续作为后续反馈标定的基础能力

不能确认：

- 反馈电压范围
- 反馈引脚接法
- 是否需要分压
- 反馈电压与实际角度之间的映射关系

## 4. 明天的测试计划

待反馈舵机到位后，按下面顺序继续：

1. 确认反馈线实际输出电压范围
2. 判断是否需要分压到 `3.3V` 以下
3. 在 `CubeMX` 中配置反馈 ADC 引脚
4. 固定若干角度点位，例如 `0 / 30 / 60 / 90 / 120 / 150 / 180`
5. 用万用表记录各角度对应反馈电压
6. 基于实测数据建立电压到角度的 mapping
7. 再补“串口命令触发后持续打印当前角度”的功能
