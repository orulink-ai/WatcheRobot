# 项目结构说明

> **日期**: 2026-04-15
> **更新日期**: 2026-04-18
> **项目**: watcheRobot_STM32
> **定位**: 当前主线代码结构、目录职责与扩展约束说明

> [!IMPORTANT]
> 本文描述的是当前仓库已提交旧实现和当前代码结构现状，不是 `proto/v2.0.0-stm32-align-20260416` 的 STM32 协处理器主线规范。
> 后续 v2 开发请先阅读 `Documents/STM32_v2文档入口.md`、`Documents/上游协议基线.md`、`Documents/分支规划.md` 和 `Documents/开发任务拆解.md`。

## 1. 当前目录职责

- `Core/`：CubeMX 生成代码、启动文件和中断入口，尽量不放业务逻辑。
- `Drivers/`：HAL/CMSIS 官方驱动。
- `User/App/`：应用编排、主循环调度、串口命令入口。
- `User/Device/`：设备驱动根目录；新模块优先按“一个芯片、一个驱动或一个聚合模块一个子目录”的形式组织代码与 README。
- `User/Platform/stm32/`：对 HAL 的轻量封装，例如 UART、I2C、时间、PWM。
- `User/Config/`：项目级常量和模块配置。
- `User/Board/`：板级引脚和硬件连接定义。
- `User/TestHost/`：后续主机侧纯逻辑测试预留目录。
- `Documents/`：项目级设计说明、硬件资料、结构文档和跨模块记录。

## 2. Core 与 User 的边界

- `Core` 负责“芯片和工程初始化”，例如时钟、GPIO、I2C、TIM、USART、中断向量。
- `User` 负责“项目自己的行为”，例如设备协议、命令解析、应用调度和打印封装。
- 当前主线入口已经收敛为：`main.c` 初始化外设，`App_Init()` 初始化业务，`App_RunOnce()` 驱动主循环。
- `main.c` 应只保留以下流程：
  - HAL 初始化
  - CubeMX 初始化
  - `App_Init()`
  - `while (1) { App_RunOnce(); }`

## 3. 新增功能应该放哪一层

- 新增串口命令：优先放 `User/App/cli.c`
- 新增设备协议：放 `User/Device/<driver_name>/`
- 设备私有文档：优先放在对应驱动目录下的 `README.md`
- 项目级或跨模块文档：继续放 `Documents/`
- 新增 HAL 访问封装：放 `User/Platform/stm32/`
- 新增全局常量：放 `User/Config/`
- 新增板级引脚定义：放 `User/Board/`

## 4. 继续使用 CubeMX 的注意事项

- 不要把业务逻辑重新写回 `Core/Src/main.c` 的普通区域。
- 如果必须在 CubeMX 文件中加自定义代码，只放在 `USER CODE BEGIN/END` 区块。
- 自定义模块优先新建在 `User/` 下，再通过 CMake 显式加入构建。
- 对设备驱动，优先采用：
  - `User/Device/<driver_name>/<driver_name>.c`
  - `User/Device/<driver_name>/<driver_name>.h`
  - `User/Device/<driver_name>/README.md`
- 重新生成工程后，优先检查：
  - `main.c` 的 `App_Init/App_RunOnce` 调用是否仍在
  - `CMakeLists.txt` 是否还包含 `User/` 目录源码
  - 新增外设初始化是否需要同步到 `User/Platform/stm32/`

## 5. 当前主线说明

- 当前主线代码已切换为“上电归中、默认静止、通过 `USART1 CLI` 设置舵机角度”的工作方式。
- 当前主线已新增 `WS2812` 调试链路，输出口径为 `TIM1_CH1 + PA8 + DMA1_Channel2`，命令入口继续收敛在 `USART1 CLI`。
- `main.c` 仍只承担 HAL/CubeMX 初始化和 `App_Init()/App_RunOnce()` 调度，串口命令解析与舵机控制逻辑继续收敛在 `User/App/`。
- 当前 `servo`、`ws2812`、`ip5306`、`bmi160`、`qmc6309`、`sensors` 与 `touch_sensor` 已按子目录组织，模块说明同步内聚到各自目录下的 `README.md`。
- 当前板间通信接口仍使用 `USART2`，引脚为 `PA2(TX)` 和 `PA3(RX)`。
- 当前 `USART2_TX` 使用 `DMA1_Channel7`，`USART2_RX` 使用 `DMA1_Channel6`，并已打通 `RX DMA + IDLE` bring-up 路径。
- 当前已引入 `User/Protocol/` 协处理器协议核心和 `WATCHER_STRESS_BUILD` 压测模拟器。
- 舵机反馈读取尚未进入当前主线：`ADC/CubeMX` 尚未配置，反馈采样、映射与串口打印待后续补齐。
- 当前 `WS2812` 已支持静态设色、活动灯珠数量调整和整条呼吸效果，但硬件联调结果仍以接线后的实测为准。

## 6. 串口与 DMA 同步约束

- `USART1` 继续承担调试打印和本地命令行交互。
- `USART2` 预留给身体板与 Watcher 头部板之间的通信。
- 如果后续继续在 CubeMX 中调整 `USART2`、DMA 或中断，请同步检查：
  - `Core/Src/usart.c` 中的引脚和 DMA 绑定是否符合实际接线
  - `Core/Src/dma.c` 与 `Core/Src/stm32f1xx_it.c` 是否生成了对应中断入口
  - `README.md` 中的引脚说明、波特率和 DMA 说明是否仍然正确
