### 4.4 时间基控制的有效精度

当前时间基运动不再按整数角度逐步发送，而是在 STM32 端根据目标角度换算到目标脉宽，并在 `motionStartPulse -> motionTargetPulse` 之间做非阻塞 us 插值。

关键边界如下：

| 约束 | 当前值 | 含义 |
|------|--------|------|
| TIM3 计数精度 | 1μs | `PSC=72-1`，72MHz / 72 = 1MHz |
| PWM 周期 | 20ms | 50Hz 标准舵机信号周期 |
| 脉宽范围 | 500~2500μs | 2000μs 对应约 180° |
| 理论命令分辨率 | 约 0.09°/μs | 180° / 2000μs |
| 手册死区 | 3μs | 小于该量级的变化可能不会立刻产生机械响应 |
| 有效机械边界 | 约 0.27° | 3μs × 180° / 2000μs |

`servo_move_time` 的 `steps` 参数用于请求运动更新密度，当前 CLI 限制为 `1~3000`。固件会先换算为更新周期，再受 20ms PWM 周期约束：

```
requestedPeriodMs = round(durationMs / steps)
effectivePeriodMs = max(20ms, requestedPeriodMs)
effectiveSteps    = ceil(durationMs / effectivePeriodMs)
pulsePerStepUs    = abs(targetPulse - startPulse) / effectiveSteps
```

`ease` 曲线支持可选 `ease_strength`，范围 `0~100`，默认 `100` 以兼容旧命令。固件实现不是浮点贝塞尔，而是把线性进度 `t` 与 smoothstep `3t²-2t³` 按强度混合：

```
progress = t + (smoothstep(t) - t) * ease_strength / 100
```

因此：

- `0%`: 等价于线性曲线，峰值速度系数 `1.0`。
- `50%`: 介于线性与完整缓入缓出之间，峰值速度系数约 `1.25`。
- `100%`: 当前完整 smoothstep，峰值速度系数约 `1.5`。

判断原则：

- `peakPulsePerFrameUs < 3μs`: 峰值仍低于舵机死区，可能出现攒误差后跳动，继续增大 steps 收益有限。
- `3~8μs`: 较适合慢速观察，峰值速度受控且仍大于死区。
- `8~12μs`: 正常可用，可能有轻微阶梯感。
- `>12μs`: 单帧峰值跨度偏粗，慢速动作可能不够柔和。

Web 调试页的“自动细腻”模式会根据当前命令角度、目标角度和曲线强度自动反推 `durationMs` 和 `steps`，默认控制峰值为 `4.2μs/frame`：

```
deltaPulseUs = abs(targetPulse - startPulse)
peakFactor = 1.0 + ease_strength / 200   // linear 固定为 1.0
peakWeightedDeltaUs = deltaPulseUs * peakFactor
minUsefulSteps = ceil(peakWeightedDeltaUs / 8us)
maxUsefulSteps = floor(peakWeightedDeltaUs / 3us)
effectiveSteps = clamp(ceil(peakWeightedDeltaUs / targetPeakUsPerFrame), minUsefulSteps, maxUsefulSteps)
durationMs = effectiveSteps * 20ms
steps = effectiveSteps
```

该模式仍然发送 `servo_move_time` 命令，只是在 `ease` 曲线后追加可选强度参数；旧命令不追加强度时仍按 `100%` 执行。
调试页计划曲线会同时绘制连续曲线和每个 20ms 有效帧点，用于观察线性与缓入缓出在实际离散更新下的步进差异。

例子：Servo1 从 150° 下降到 120°，目标脉宽约 `2167μs -> 1833μs`，总变化约 `333μs`。

| 曲线 | 强度 | 时长 | 有效帧数 | 平均每帧 | 峰值每帧 | 备注 |
|------|------|------|----------|----------|----------|------|
| 线性 | - | 1600ms | 80 | 约 4.2μs | 约 4.2μs | 峰值与平均一致 |
| 缓入缓出 | 50% | 2000ms | 100 | 约 3.3μs | 约 4.2μs | 比线性更柔，峰值仍受控 |
| 缓入缓出 | 100% | 2400ms | 120 | 约 2.8μs | 约 4.2μs | 起止最柔，中段速度最高 |
| 缓入缓出 | 100% | 3000ms | 150 | 约 2.2μs | 约 3.3μs | 更慢，但端点可能接近死区 |

因此，当前 TIM3 的 1μs 精度已经细于舵机 3μs 死区；继续提高定时器分辨率可以让数字更细，但优先收益通常不如优化运动曲线、供电和机械负载。

### 4.5 面向动作设计的线性动作规范

当前动作设计默认优先使用 `linear`。线性曲线的好处是每个 20ms PWM 有效帧的脉宽变化基本一致，方便把变化量稳定控制在舵机可响应的死区之外。

动作设计师需要注意：Blender 里的动画是连续插值，时间拉长通常会看起来更丝滑；真实舵机不是连续画面，而是每 20ms 接收一次 PWM 脉宽。如果每一帧的脉宽变化太小，舵机会把它当成死区内的微小误差，前几帧不动，攒到一定量后再突然跳一下。视觉上就会表现为末尾粘住、抖动或一顿一顿。

推荐把线性动作控制在 `4~8μs/frame`。换算公式如下：

```
deltaPulseUs = angleDeltaDeg * 2000us / 180deg
effectiveFrames = durationMs / 20ms
pulsePerFrameUs = deltaPulseUs / effectiveFrames
```

按 `4~8μs/frame` 反推常用动作时长：

| 角度变化 | 脉宽变化 | 推荐到达时间 | 说明 |
|----------|----------|--------------|------|
| 10° | 约 111μs | 280~560ms | 小动作，太慢容易落入死区 |
| 20° | 约 222μs | 560~1120ms | 常用轻微点头/摆头 |
| 30° | 约 333μs | 840~1680ms | 常用明显转头/低头 |
| 40° | 约 444μs | 1120~2240ms | 大动作，建议先用实物复核负载 |

例子：如果 Blender 里设计了一个 30° 的慢低头，持续 3000ms，看起来会很柔和；但实物中 30° 约等于 333μs，3000ms 有 150 个有效帧：

```
333us / 150frames = 约 2.2us/frame
```

这低于推荐死区边界，实物可能不是更细腻，而是后段抖动或突然补动。把同一个 30° 动作控制在 840~1680ms 内，每帧约 8~4μs，更容易获得稳定的物理响应。

#### 抬头角度限制

基于当前实测，抬头目标角度建议不要超过 `130°`。超过 130° 后，即使使用较细的时间基控制，后续低头回落时仍容易出现抖动。这更像机械负载、重心和舵机保持力共同造成的物理边界，不应仅靠继续增加 steps 或延长时间解决。

动作设计建议：

- 常规抬头动作目标控制在 `90~130°`。
- 需要表现“极限抬头”时，先用实机验证，不要默认超过 `130°`。
- 从高角度低头回落时优先使用 `linear`，并按上表保持 `4~8μs/frame`。
- 如果动作仍抖，优先降低最高抬头角度，再检查供电、连杆阻力和舵机负载。

## 5. API 使用指南

### 5.1 当前主线 API: 命令驱动模式初始化

```c
// 定义配置结构体
static const Servo_ConfigTypeDef s_servo1Config = {
    .pTimHandle     = &htim3,
    .TimChannel     = TIM_CHANNEL_1,
    .pulseMin       = 500,        // 0°位置
    .pulseMax       = 2500,       // 180°位置
    .pulseCenter    = 1500,       // 90°中心位置 (初始化默认)
    .stepSize       = 30,         // 步进量 (μs) - 越小越平滑
    .stepDelayMs    = 20          // 步进间隔 (ms) - 越小越快
};

// 初始化并获取句柄
Servo_HandleTypeDef hServo1 = Servo_Init(&s_servo1Config);
// 初始化后舵机会自动归中
```

### 5.2 当前主线 API: 命令驱动控制

```c
uint16_t pulseUs = 0U;

// 通过 App 层封装直接设置目标角度

    // pulseUs 将返回当前角度对应脉宽，例如 1500us
}

// 主循环保持状态更新和 CLI 轮询

    App_RunOnce();

```

### 5.7 当前与历史 API 的边界

- `Servo_StartReciprocating()` / `Servo_StopReciprocating()` 仍保留在驱动层，用于兼容旧的往复运动状态机
- 当前主线默认不会在 `App_Init()` 中启动往复运动
- 一旦通过 CLI 调用 `App_SetServoAngle()`，对应舵机会先停止往复模式，再直接写入目标角度

如果需要使用其他定时器通道或不同定时器：

1. 在 `tim.c` 中配置对应的定时器和通道
2. 在 `User/App/app.c` 中定义新的配置结构体
3. 在 `App_Init()` 中调用 `Servo_Init()` 初始化

示例 (使用 TIM2_CH1，假设已在 CubeMX 中配置):

```c
static const Servo_ConfigTypeDef s_servo3Config = {
    .pTimHandle     = &htim2,     // TIM2
    .TimChannel     = TIM_CHANNEL_1,
    .pulseMin       = 500,        // 0°位置 (数据手册下限)
    .pulseMax       = 2500,       // 180°位置 (数据手册上限)
    .pulseCenter    = 1500,       // 90°中心位置
    .stepSize       = 66,
    .stepDelayMs    = 30
};
```

### 6.2 常见舵机类型

| 类型 | 脉宽范围 | 角度范围 | 典型用途 |
|------|----------|----------|----------|
| 标准舵机 | 1000-2000μs | 0-180° | 机械臂、机器人关节 |
| 连续旋转舵机 | 1000-2000μs | 速度控制 | 轮子驱动 |
| 微型舵机 | 900-2100μs | 0-120° | 航模、小型项目 |

### 6.3 注意事项

1. **供电**: 舵机需要足够的电流供应，STM32 引脚无法直接驱动大功率舵机
2. **电压匹配**: 确保舵机供电电压与规格匹配 (通常 4.8-6V)
3. **信号干扰**: 长线缆可能引入干扰，建议使用屏蔽线
4. **初始化顺序**: 确保 `MX_TIM3_Init()` 先于 `Servo_Init()` 执行

### 6.4 舵机速度控制原理

#### PWM 与角度的关系

舵机角度由 **PWM 信号的脉宽（高电平持续时间）** 决定，周期固定为 20ms (50Hz)：

```
┌──────────────────────────────────────────────────────────┐
│                     20ms 周期 (50Hz)                      │
│                                                          │
│   500μs ─────────────┤                      ├─ 2500μs   │
│        (0°)         │                      │   (180°)  │
│                     │                      │            │
│        1500μs ──────┼──────────────────────┼─ 1500μs   │
│         (90°)       │      中心位置         │   (90°)    │
└──────────────────────────────────────────────────────────┘
```

#### 定时器 CCR 寄存器原理

```
计数器 CNT ──────►
              ┌──┐     ┌──┐     ┌──┐
PWM 输出  ──┘      └──┘     └──┘     └──┘
            ▲      ▲      ▲
            │      │      │
           CCR1   CCR2   CCR3

当 CNT < CCR 时输出高电平，CNT > CCR 时输出低电平
通过写入不同的 CCR 值，硬件自动生成不同占空比的 PWM
```

#### 速度是如何实现的

在往复运动状态机中 ([servo.c](./servo.c))，每次 `Servo_Update()` 调用时：

```c
// 每次调用只迈进一步
pInst->state.currentPulse += pInst->config.stepSize;  // 步进量
Servo_WriteCCR(&pInst->config, pInst->state.currentPulse);  // 写入硬件
```

状态机会按 `stepDelayMs` 节奏生效一次步进，速度计算公式：

```
速度 = stepSize / stepDelayMs

例如 PA6 (stepSize = 66μs, stepDelayMs = 30ms):
  - 每步脉宽增量: 66μs
  - 每步时间: 30ms
  - 每秒脉宽增量: 66μs × (1000ms / 30ms) = 2200μs/s
  - 对应角度速度: 2200μs/s × (180° / 2000μs) ≈ 198°/s
```

#### 速度差异的奥秘

两个舵机速度不同，根本原因是 `stepSize` 不同：

| 参数 | PA6 (快) | PA7 (慢) | 说明 |
|------|----------|----------|------|
| stepSize | 66μs | 10μs | 每步脉宽增量 |
| 每方向步数 | 约 31 步 | 200 步 | 到达极端所需步数 |
| 往返耗时 | 约 1.86秒 | 12秒 | 完成一次往复 |
| **本质差异** | **大步长** | **小步长** | 速度与步长成正比 |

**小步长 = 更精细的控制 = 更慢的速度**

#### 与传统 for 循环方式的对比

| 特性 | for 循环方式 | 状态机方式 |
|------|-------------|-----------|
| **结构** | 循环内部完成整个运动 | 每次调用只走一步 |
| **阻塞** | 整个运动期间阻塞主循环 | 每步执行后返回主循环 |
| **多舵机** | 难以实现（会互相阻塞） | 可同时控制多个舵机 |
| **暂停/恢复** | 需要重置整个循环 | 只需改 flag |
| **速度控制** | 同样通过 stepSize | 同样通过 stepSize |

**本质上速度控制原理是一样的** — 都是 `脉宽增量 / 时间间隔`。区别在于：
- for 循环：**连续**改变脉宽，阻塞式
- 状态机：**分步**改变脉宽，非阻塞式
